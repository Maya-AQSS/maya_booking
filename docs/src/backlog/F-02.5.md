# [BACKEND] F-02.5 — Liberación Masiva de Espacios por Fin de Estudios

## 🏷️ Capa Técnica
**Capa:** `backend`
**Impacta en:** Equipo de Desarrollo Odoo

**Prioridad:** `Could`

---

**Descripción:**
Como Directiva, quiero poder cancelar reservas masivamente a partir de una fecha límite por fin de curso.

**Criterios de Aceptación (Checklist):**
- [ ] **Escenario 1 (Happy Path):** Acción masiva que trunca la fecha tope de un conjunto de reservas.
- [ ] **Escenario 2 (Edge Case):** Libera los huecos para otros usuarios.
- [ ] **Requisito Técnico:** Restricción de base de datos o lógica server-side gestionada vía ORM/Vistas en Odoo.
