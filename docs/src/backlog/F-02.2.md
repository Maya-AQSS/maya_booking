# [BACKEND] F-02.2 — Lógica de Solapamientos y Repetición Múltiple

## 🏷️ Capa Técnica
**Capa:** `backend`
**Impacta en:** Equipo de Desarrollo Odoo

**Prioridad:** `Must`

---

**Descripción:**
Como Profesor, quiero reservar periodos repetitivos sin que el sistema permita solapamientos con otras reservas.

**Criterios de Aceptación (Checklist):**
- [ ] **Escenario 1 (Happy Path):** El sistema bloquea reservas cuando coinciden un solo minuto.
- [ ] **Escenario 2 (Edge Case):** Se crean registros semanales hasta la fecha tope indicada.
- [ ] **Requisito Técnico:** Restricción de base de datos o lógica server-side gestionada vía ORM/Vistas en Odoo.
