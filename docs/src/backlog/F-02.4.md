# [FULLSTACK] F-02.4 — Cancelación y Modificación con Notificaciones

## 🏷️ Capa Técnica
**Capa:** `fullstack`
**Impacta en:** Equipo de Desarrollo Odoo

**Prioridad:** `Must`

---

**Descripción:**
Como usuario o Directiva, quiero modificar/cancelar reservas enviándose un correo automático si es modificado por un tercero.

**Criterios de Aceptación (Checklist):**
- [ ] **Escenario 1 (Happy Path):** Usuario puede anular las suyas.
- [ ] **Escenario 2 (Edge Case):** Equipo directivo anula las de un profesor y este recibe un email.
- [ ] **Requisito Técnico:** Restricción de base de datos o lógica server-side gestionada vía ORM/Vistas en Odoo.
